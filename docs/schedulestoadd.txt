================================================================================
QStash: CONSOLIDATED CRON SCHEDULES (5 total)
================================================================================
Use only these 5 schedules to stay under QStash limits and minimize messages.
Each dispatcher runs multiple jobs internally (1 QStash message per run).

Auth for all: Header  X-Internal-Api-Key: YOUR_INTERNAL_API_KEY
(same value as INTERNAL_API_KEY in backend .env).

Base URL: https://YOUR_BACKEND_HOST   (e.g. https://api.deptex.dev)

--------------------------------------------------------------------------------
1. EVERY 5 MINUTES  (recovery + Aegis automations)
--------------------------------------------------------------------------------
Cron:  */5 * * * *
URL:   POST https://YOUR_BACKEND_HOST/api/internal/cron/every-5-min

Runs:  extraction-jobs recovery, fix-jobs recovery, watchtower-jobs recovery,
       Aegis check-due-automations.

--------------------------------------------------------------------------------
2. EVERY 15 MINUTES  (notifications + SLA)
--------------------------------------------------------------------------------
Cron:  */15 * * * *
URL:   POST https://YOUR_BACKEND_HOST/api/internal/cron/every-15-min

Runs:  reconcile-stuck-notifications, sla-check (EE).
       (In CE mode these return 404 and are skipped.)

--------------------------------------------------------------------------------
3. HOURLY
--------------------------------------------------------------------------------
Cron:  0 * * * *
URL:   POST https://YOUR_BACKEND_HOST/api/internal/cron/hourly

Runs:  vuln-check, reset-sync-counters, learning check-feedback-prompts,
       digest-check (EE).

--------------------------------------------------------------------------------
4. DAILY at 4 AM UTC
--------------------------------------------------------------------------------
Cron:  0 4 * * *
URL:   POST https://YOUR_BACKEND_HOST/api/internal/cron/daily

Runs:  Aegis snapshot-debt, learning recompute-patterns,
       notification-cleanup (EE), watchtower-daily-poll.

--------------------------------------------------------------------------------
5. EVERY 6 HOURS
--------------------------------------------------------------------------------
Cron:  0 */6 * * *
URL:   POST https://YOUR_BACKEND_HOST/api/internal/cron/every-6h

Runs:  scheduled-extraction (daily/weekly sync_frequency projects).

================================================================================
MESSAGES PER DAY (approx)
================================================================================
  every-5-min:  288
  every-15-min:  96
  hourly:        24
  daily:           1
  every-6h:       4
  -----------------
  Total:        413  (vs 10+ separate schedules with more messages if each had its own cron)
