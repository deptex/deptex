## Overview

Make the taint/EPD pipeline score findings by the *real* auth exposure of their entry points instead of hardcoding every flow to `PUBLIC_UNAUTH` (`taint-engine/storage.ts:275`). Three legs: (1) upgrade all 34 framework detectors from file-level import sniffing to **route-level auth evidence** with **handler spans**; (2) stamp each engine flow from the route whose handler span contains the flow's source line (fail-safe: no span match → PUBLIC weight, no merge vote); (3) merge deterministic evidence with the per-flow Qwen verdict in EPD and surface via `EntryPointBadge`. Zero migrations; reachability untouched.

## Competitive Research & Design Rationale

(Condensed — citations in brief.) Static middleware-derived auth exposure wired into scoring is whitespace (Snyk/Endor source "public facing" from runtime agents; [route-detect](https://github.com/mschwager/route-detect) proves per-framework classification tractable but isn't wired to scoring).

## Core Semantics (the contract)

1. **Evidence-based classification.** `AUTH_INTERNAL`/`OFFLINE_WORKER` only from (a) route-level evidence (middleware arg, decorator, annotation, fluent chain, DI dependency) or (b) a Locked-5 centralized idiom positively covering that route. Import sniffing never classifies; `authMechanism` keeps the import hint (DAST depends on it).
2. **Explicit-public overrides win** (pattern-based): `@PermitAll`, `[AllowAnonymous]`, `.AllowAnonymous()`, `@Public()`, `skip_before_action`, DRF `AllowAny`/`permission_classes=[]`/`authentication_classes=[]`, `@login_not_required`, `->withoutMiddleware('auth*')`, `IsGranted('PUBLIC_ACCESS'|'IS_AUTHENTICATED_ANONYMOUSLY')`, SpEL `permitAll|isAnonymous|true` in `@PreAuthorize`, + name pattern `public|anon(ymous)?|skip.?auth|no.?auth|unprotected|allow.?unauth`. Method-level public beats class/group auth.
3. **Conditional or carve-out coverage = not covering.** Rails `before_action` with ANY kwarg except fully-resolved matching `only:`; Spring `SecurityFilterChain` only with zero `permitAll()/anonymous()/webIgnoring` carve-outs; DRF `IsAuthenticatedOrReadOnly`; `.unless(` anywhere in a matched middleware chain; Sinatra `before '<pattern>'`; Next.js matchers.
4. **Optional/parse-only auth is never evidence.** Veto tokens `optional|anonymous|guest|maybe|try|soft` on identifier/args; `credentialsRequired:false`, `auto_error=False`, passport `'anonymous'` (incl. arrays); parse-vs-enforce (chi `jwtauth.Verifier` ≠ `Authenticator`). Ruby: bang-convention/known-halting allowlist (`authenticate_user!`, `authenticate_admin!`, `require_login`, `authorize!`); bare `authenticate` is not evidence.
5. **Machine evidence → OFFLINE_WORKER** only from: verifier-call shapes **in the route's own file** (`<recv>.verify(` on Receiver/Webhook instances, `stripe.webhooks.constructEvent(`) or chain middleware matching `internal|signature|hmac|webhook.?verif|qstash|svix`. Bare SDK usage never counts (svix send ≠ verify); **route path/name strings never count** (unverified `/webhooks/github` stays PUBLIC — negative test). ★Cross-file verifier bodies (Stripe/GitHub-style lib handlers) are NOT detectable in v1 → those endpoints stay PUBLIC (documented residual, T13(c) expects it).
6. **Join v3 = handler-span containment (replaces v2's demotionSet entirely).**
   - **Span-node rule (R3-pinned, r3-ec-1):** `handlerSpan` is the span of the **terminal handler-argument node exclusively** — never the registration call node, never any non-terminal (middleware) argument. Rationale: middleware args before the auth arg run pre-auth and read request data; a span over the whole registration would demote pre-auth sources. Sources firing in inline middleware bodies (pre- OR post-auth) stamp **unmatched** (safe coverage loss, no ordering analysis).
   - **Line convention (R3-pinned, ts3-f1):** span lines are **1-based, inclusive both ends**, computed via the existing `lineOf()` (`startPosition.row+1` / `endPosition.row+1`, `util/javascript.ts:130`); containment = `startLine <= entry_point_line <= endLine` (matches flow `entry_point_line`, 1-based). One-line arrow handlers have `startLine === endLine`.
   - **Span source per route:** inline function/arrow/block handler → that terminal-arg node's span. Named handler → resolve **only** when the terminal-arg node is a **bare `identifier`** resolving to exactly ONE same-file `function_declaration` OR a `variable_declarator` whose value is directly a function/arrow expression → that declaration's span; **`member_expression` (`ctrl.method`), `call_expression` (wrapped `asyncWrap(h)`), array-form, and cross-file/unresolvable handlers → null span** (r3-ec-4). Mount/`use`/config rows carry no span. Rails/Django postProcess computes **per-action/per-view spans** from banked authFacts for routes resolved from explicit verb declarations (`resources`-routed → span-less residual, follow-up).
   - **Demotion eligibility — per-language guard (R3 convergent, r3-ec-3/skeptic3-f2/arch3-1/ts3-f5).** A span is *demotion-eligible* unless the handler could be invoked from code we can't see. The guard applies **per family**:
     - **Reference-registered families (JS/TS, Go):** a *named* handler is **ineligible** if it could be re-mounted/called elsewhere. JS/TS: ineligible iff its identifier appears in **any export-shaped construct in the file** (structural over-approximation, not a literal list — any `export_statement` wrapping its declaration; any export clause naming it incl. `export {h}` / `export {h as x}` / `export default h`; any assignment whose LHS member chain roots at `exports`/`module.exports` either side; TS `export =`) **OR** the identifier is **referenced anywhere in the file outside its own direct-handler-argument registration position** (r3-ec-4/skeptic3-f3 — catches same-file double-registration & wrapped re-mounts like `router.get('/pub', rateLimit(admin))`). Go: package is the privacy unit, so **capitalized identifier is not sufficient privacy** — ineligible iff the handler name appears in any *other* same-package file's route-registration facts (cheap name scan over banked facts at T2 build) OR is referenced outside its registration in its own file. **Inline (anonymous) handlers are always eligible** (cannot be referenced elsewhere).
     - **Declaration-bound families (nestjs, flask, fastapi, spring, jaxrs, quarkus, micronaut, symfony, aspnet-core, minimal-apis, laravel, slim, actix, rails/django postProcess actions):** guard **N/A → always eligible.** The auth evidence is syntactically AT the declaration (decorator/annotation/attribute/action), so it travels with the handler wherever it's referenced; there is no "registered here, defined there with different auth" split. (This is why the guard is not a global rule — applying JS export-semantics to these would guard every public method/function and gut the leg.)
   - **Stamping (taint-regime flows only):** candidates = routes in `entryPointAuth[entry_point_file]` whose span contains `entry_point_line`. ≥1 candidate → worst-case across candidates: (all candidates authed/internal AND demotion-eligible) → `framework-route:<class>`; any candidate evidence-PUBLIC → `framework-route:public_unauth`; any candidate UNKNOWN, or any candidate authed-but-**ineligible**, and none evidence-public → `framework-input:unmatched`. 0 candidates → `framework-input:unmatched`. Detector-coerced flows keep the legacy constant.
   - Consequences (why this is sound): flows in helpers/services/exported-or-referenced handlers/alias-registered/`resources`-routed/unseen routes are **unmatched → PUBLIC weight, no vote** — every R1/R2/R3 wrongful-demotion shape (exported helper, barrel, wrapper, same-file double-registration, alias, mixed `.all()` file, inline `/health` neighbor, mixed controller, pre-auth middleware body) is structurally fail-safe; no export *extraction*, no module resolution, no httpMethod discriminator.
7. **Merge (R3-pinned, skeptic3-f4/arch3-4).** The aggregator (`epd.ts aggregateEpdFromFlows`) splits its single filter into **two vote sets**:
   - `sanitizationVoters` = the **existing** predicate **byte-for-byte unchanged** (`!suppressed && filterVerdict∉{ai_truncated,kept_on_error} && sanitization!==null && endpoint!==null && confidence>=MAX_VOTE_THRESHOLD`); `allSanitized` is computed over this set ONLY (no behavior change to sanitization).
   - `endpointVoters` = `!suppressed && filterVerdict∉{ai_truncated,kept_on_error}` flows, each contributing `effective = maxRisk(AI endpoint verdict if present, matched `framework-route:` evidence)`. Matched evidence can **never override an AI-public verdict** (Locked-6). `unmatched`/legacy tags contribute nothing; a verdict-less flow contributes matched evidence alone.
   - **Direction guard (the intended-expansion decision):** relative to today (where sub-threshold flows don't vote at all), a verdict-less **evidence-authed** flow can now demote where it previously wouldn't. That is intended (deterministic evidence is higher-confidence than a missing AI verdict) and **fail-safe in direction** (demotion only). The T4 matrix pins the AI-public-beats-evidence-authed case and the verdict-less-evidence-authed-demotes case explicitly.
   - `no_flows_evaluated` fallback: worst-case across non-suppressed `taint_engine` flows' tags (legacy-source rows keep current shortest-flow behavior). `all_flows_suppressed` unchanged. `epd_confidence_tier`/`flowsAggregated` restated to describe the **endpointVoters** set that actually decided the class (arch2-6/arch3-4).
8. **Auth-name patterns** (word-boundary, case-insensitive, matched on the **import-resolved** symbol, literal fallback): `authenticate|authenticat(ed|or)|require(s)?Auth|ensure(Auth|Logged)|isAuthenticated|login_?required|jwt_?required|verifyToken|checkAuth|checkJwt|passport\.authenticate|expressjwt|requireUser|authorize` + per-framework additions, subject to Semantics-4 vetoes. NestJS: `@UseGuards` argument must match (`*AuthGuard`); ThrottlerGuard neutral.
9. **DAST (decided):** OFFLINE_WORKER exclusion from the synthesized spec is intended (signature endpoints yield unauthenticated-scan noise); asserted in T13. Name-pattern-only OFFLINE_WORKER evidence costs DAST coverage on a false match — watched via the T13(a) allow-list + the express negative fixtures.
10. **Public-route-name belt** (owned by T1's `classifyRoute`, centralized-coverage path only): routes whose pattern contains a **path segment** matching `login|logout|signin|signup|register|password|health|status|ping|webhook|callback|oauth|\.well-known` never inherit a *centralized* demotion (route-local evidence may still demote). Segment-boundary matching (`/users/:id/login` matches; `/loginit` doesn't). Belt fixtures required (T1).
11. **Prefix coverage is path-segment-bounded:** `app.use('/api', mw)` covers `/api` and `/api/…`, never `/apiv2` (skeptic2-f4). Same rule for group prefixes everywhere.

## Risks & Open Questions

- **Span capture per detector** is new per-framework work, but strictly simpler than v2's veto machinery: inline spans are free (node positions), same-file named resolution is one lookup, export detection is syntactic. Residual: flows via handlers we can't span (wrapped/cross-file/`resources`) are unmatched → PUBLIC — a **coverage** cost, not a soundness cost; measured by the join-coverage KPI and prioritized by it (os2-r2-f1).
- **Residual wrongful-demote shape:** an authed route's *non-exported, same-file* handler invoked directly from public code elsewhere. Requires calling another route's handler as a plain function without export — rare; accepted + documented.
- **Rails `resources` blindness** makes the Rails leg partial on resources-heavy apps (discourse/mastodon) — honest residual, follow-up recorded; T13(a) distributions make the coverage visible.
- **T5 prompt drift on AI-on orgs** — bounded by T13(d) + prompt tests; corpus runs filter-off (documented).
- Deferred (recorded): resources expansion, cross-file verifier bodies (Stripe/GitHub webhook OFFLINE_WORKER), express mount-graph, nextjs guard-shape, rocket/warp, tornado cross-file, evidence-string badge, policy/x-deptex/Aegis/attack-surface-view exposures.
